# Чеклисты

## Перед ответом на вопрос

- [ ] Загружен минимум 1 релевантный файл
- [ ] Определён тип репозитория (Pack/Downstream/...)
- [ ] Ответ опирается на контекст репозитория
- [ ] Источники указаны
- [ ] **Pack direct read:** При поиске конкретного знания в Pack — Read наиболее вероятный файл напрямую, не только vector search (индекс может быть stale)

## Перед изменением репозитория

- [ ] Прочитан CLAUDE.md репозитория (если есть)
- [ ] Понятна роль этого репо в архитектуре
- [ ] Изменения не нарушают source-of-truth

## При перемещении / переименовании репозитория

- [ ] `grep -r` по ВСЕМ репо — найти ВСЕ внутренние ссылки
- [ ] Скрипты (.sh), промпты (.md), конфиги (.yaml), plist, GH Actions (.yml), README, CLAUDE.md
- [ ] Загруженные plist в `~/Library/LaunchAgents/` (отдельно от шаблонов!)

## При ручной очистке fleeting-notes (🔄 заметки)

- [ ] Для каждой 🔄 заметки: покрыт ли intent результатом РП? Да → архивировать

## При записи процесса/решения о системе

- [ ] **Pack** + **PROCESSES.md** + **CLAUDE.md** (если кросс-системное). Процесс системы = часть паспорта.

## Перед созданием документа

- [ ] Спросить пользователя — предложить место с обоснованием (репо + путь + почему)
- [ ] Frontmatter + связи с другими документами

## Перед предложением фикса / архитектурного решения

- [ ] АрхГейт ДО ответа (в первый ответ, не после запроса)
- [ ] Существующие паттерны: «как это уже решено в кодовой базе?»
- [ ] Корневая причина, не симптом
- [ ] Итеративные заплатки: если 2-й коммит «чинит» 1-й → СТОП

## После написания поста (DS-Knowledge-Index)

- [ ] `docs/README.md` обновлён
- [ ] Закоммичено и запушено

## Close (дополнение к protocol-close.md)
